## Why Another Beauty SDK?

Most camera and photo-editing products do not fail because they lack filters. They fail because the final image is not stable enough across real user photos.

Real-world images are messy:

- indoor lighting is mixed and unpredictable;
- stage screens often cast strong blue, green, or purple light;
- group photos need background enhancement without destroying skin tones;
- beauty effects can easily become blurry, plastic, or over-processed;
- preview and export should look consistent.

Chroma Beauty SDK is designed around this problem. It combines **AI one-tap color grading**, **portrait-safe image enhancement**, **beauty retouching**, **face shaping**, and **preview/export workflow** into one mobile-oriented imaging pipeline.

## Demo App Workflow

![Chroma Beauty SDK Demo Flow](https://raw.githubusercontent.com/18818474455/ChromaBeautySDK/main/assets/apk-demo/demo-flow.gif)

The demo workflow is simple:

```text
Import image
-> Run AI color recommendation
-> Keep portrait protection enabled
-> Fine-tune base, detail, color, style and beauty parameters
-> Compare with original image
-> Export the final result
```

This is not just a filter picker. The goal is to give product teams a complete editing experience that can be integrated into a real mobile app.

## 1. AI Color Grading That Still Feels Editable

![AI One-Tap Color Grading](https://raw.githubusercontent.com/18818474455/ChromaBeautySDK/main/assets/apk-demo/one-tap-color.jpg)

One-tap enhancement is useful only when users can still control the result.

Chroma can generate a recommended color style and expose the result as adjustable parameters, including exposure, contrast, highlights, shadows, color temperature, tint, saturation, HSL-style color control, curves, LUT-style looks, and global strength.

For product teams, this means:

- the app can provide an instant result for casual users;
- advanced users can still fine-tune the final image;
- presets and manual sliders can share the same processing pipeline;
- the experience feels closer to a modern editor than a simple filter list.

## 2. Portrait-Safe Enhancement

Global color enhancement often makes the background look better but damages the person in the photo. Skin can become too red, too yellow, too gray, or too saturated.

Chroma Beauty SDK is built with portrait protection in mind. The SDK can be used to enhance atmosphere while keeping the person visually natural.

Typical use cases include:

- event photos with strong LED screens;
- indoor selfies under warm lighting;
- group photos in malls, weddings, conferences, and live events;
- social profile photos and dating apps;
- e-commerce portraits and model images.

## 3. Beauty Retouching Without Turning the Image Into Plastic

![Beauty and Face Shaping Controls](https://raw.githubusercontent.com/18818474455/ChromaBeautySDK/main/assets/apk-demo/beauty-shape-controls.jpg)

Good beauty processing should not simply blur the entire image. A commercial beauty pipeline needs to protect edges, hair, clothing, eyes, and background details.

Chroma Beauty SDK can provide modules such as:

| Module | Product Value |
|---|---|
| Skin smoothing | Cleaner skin while keeping a natural visual impression |
| Whitening | Brighter skin tone with softer transitions |
| Rosy tone | Healthier portrait color without over-saturation |
| Shine removal | More controlled highlights on forehead, nose and cheeks |
| Dark circle correction | More polished portrait appearance |
| Face shaping | Face, eyes, nose, mouth and facial proportion adjustment |
| Body reshape | Slimming and long-leg style product features |

The important part is not only the effect itself, but the ability to expose these effects as product-friendly controls.

Chroma can be evaluated as a complete workflow or as independent modules.

### Full Workflow Mode

For teams building a camera, photo editor, or portrait retouching product from the ground up:

```text
Image input -> AI analysis -> One-tap color -> Beauty controls -> Compare preview -> Export
```

### Modular Mode

For apps that already have their own editor UI:

```text
Only integrate AI color grading
Only integrate portrait protection
Only integrate beauty retouching
Only integrate face/body shaping
Only integrate LUT, HSL or manual color controls
```

This makes the SDK suitable for different stages of product development, from MVP evaluation to commercial deployment.
